## Next Task: Themes as Modules

The user wants to **brainstorm themes as plugins/modules** for DOCSight. This is a new feature — NOT part of the settings redesign.

### Context for Brainstorming
- DOCSight's module system already supports `type: "theme"` in `manifest.json` (icon mapping exists in `_module_card.html`)
- But no actual theme module exists, and no backend logic applies theme modules
- The Module Store (`settings#modules`) currently only shows built-in modules — no marketplace/registry
- The settings redesign introduced the Tribu design system (glass morphism, void-dark palette) as a hard-coded CSS theme
- The user wants themes to be installable/toggleable like other modules

### Key Files
- Module system: `app/modules/` (each module has `manifest.json`)
- Module loader: `app/module_loader.py`
- Module card template: `app/templates/settings/_module_card.html`
- Current CSS: `app/static/css/settings.css` (1245 lines, Tribu design)
- Main CSS: `app/static/css/main.css` (dashboard + shared styles)
